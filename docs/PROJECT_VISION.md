# Project Vision

## What is Muralis?

Muralis is a local-first visual organizer for ideas, tasks, goals, references, links, quotes, and thought structures.

It lets users view the same blocks of information through different modes:

```txt
Free Board
Kanban
Matrix
Mind Map
Timeline
```

## Why does it exist?

Muralis exists to demonstrate and test a hypothesis:

> The native web platform is already powerful enough to build complex, useful, and beautiful applications without immediately depending on frameworks, libraries, or heavy toolchains.

This does not mean libraries are bad. It means they should be a conscious choice, not an automatic reflex.

## Motivation

Many projects start with accidental complexity:

- framework before the problem;
- dependency before the need;
- abstraction before the domain;
- build setup before the first interface;
- backend before validating the experience.

Muralis takes the opposite path:

```txt
product first
experience first
browser first
essentials first
```

## The experiment

The experiment is to build an app with:

- multiple views;
- centralized state;
- local persistence;
- drag and drop;
- zoom/pan;
- modals;
- command palette;
- dark mode;
- import/export;
- responsive UI;

mainly using:

```txt
HTML
CSS
JavaScript
DOM APIs
LocalStorage
SVG
```

## What does the project try to prove?

Muralis aims to show that it is possible to:

1. create a rich application without a framework;
2. keep a well-organized vanilla JS architecture;
3. build modern UI with structured CSS;
4. use the browser as a real application platform;
5. reduce dependencies;
6. deeply learn web fundamentals;
7. deliver value before adding complexity.

## Who is it for?

The project is useful for:

- people who want to organize ideas visually;
- frontend students;
- developers studying architecture without frameworks;
- people curious about local-first apps;
- creators who like simple tools;
- open source projects focused on fundamentals.

## What is Muralis not?

By default, Muralis is not:

- a SaaS;
- a real-time collaborative tool;
- an authentication-based app;
- a server-dependent app;
- a Notion, Trello, or Miro clone;
- a minimal demo.

It is a local visual app with real product ambition.

## Product principles

```txt
easy to open
easy to use
hard to lose context
easy to reorganize
low visual noise
user-controlled data
no unnecessary dependencies
```

## Guiding sentence

> Product complexity does not have to mean stack complexity.
